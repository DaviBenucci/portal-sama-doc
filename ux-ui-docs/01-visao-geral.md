# 01 - Visao Geral

## Objetivo

Melhorar a experiencia do Portal Sama tornando as telas mais simples, informativas e seguras.

A aplicacao deve se comportar como uma central operacional para:
- clientes;
- documentos;
- certificados digitais;
- vencimentos;
- propostas;
- contratos;
- entrada de cliente;
- gestao;
- T.I;
- administracao.

## Principios

### 1. A Home deve orientar acao
A Home deve mostrar o que o usuario precisa fazer, e nao apenas repetir atalhos do menu.

### 2. A navegacao deve respeitar permissao e departamento
O usuario nao deve visualizar rotas que nao fazem parte da rotina ou permissao dele.

### 3. Documentos devem ficar no contexto do cliente
A pagina de documentos deve ser acessada preferencialmente pelo painel do cliente.

### 4. Seguranca deve aparecer na UX
Como o Portal Sama manipula documentos empresariais, a interface deve reforcar:
- ambiente seguro;
- auditoria;
- confirmacao em acoes sensiveis;
- expiracao de links publicos;
- classificacao de confidencialidade.

## Stack visual recomendada

- React;
- TypeScript;
- Vite;
- Tailwind CSS;
- shadcn/ui;
- Radix UI;
- TanStack Query;
- TanStack Table;
- React Hook Form;
- Zod;
- Zustand;
- Lucide React;
- GSAP somente para intro e animacoes institucionais.
